# jacksonhe04.github.io

本仓库为我的个人网站项目，创建于2024年10月23日，旨在作为一个开源项目，展示我的技术积累与成长过程。当前，`main`分支中仅包含一个名为`FrontByVite`的目录，基于`Vite`构建了前端框架。项目的构建产物（`dist`目录）已部署在`gh-pages`分支，并通过GitHub Pages进行托管展示。

## 当前状态

### 1. 网站内容
目前，网站内容非常简易，并没有实质性功能，仅包含一些基础的占位页面。随着开发进度的推进，未来将逐步添加个人介绍、项目展示、博客文章等内容，充实网站的实际功能。

### 2. 技术框架
前端框架目前仍处于初期开发阶段，基于`Vite`进行构建，功能较少且结构简单。尽管如此，该框架为后续扩展功能提供了基础。随着项目的深入，我将逐步优化和扩展前端框架，使其更具灵活性和可维护性。

## 未来计划

为了探索不同的技术方案，并提升个人技能，我计划在后续开发中引入不同的技术栈，通过多种方式实现相同的功能，具体包括：

- **React项目目录**：计划添加一个基于`React`的目录，用来实现与`Vite`项目相同的前端功能。这将帮助我深入理解组件化开发模式，并利用`React`的生态来优化前端开发效率和代码的可扩展性。

- **Next.js项目目录**：此外，我还会创建一个基于`Next.js`的目录，通过服务端渲染（SSR）来提高网站的性能和SEO表现。该目录将展示如何使用`Next.js`实现与其他服务端框架（Express）相同的功能。

- **Express项目目录**：为了为网站提供服务端功能，我将添加一个基于`Express`的项目目录，用来处理后端逻辑。这个服务端项目将负责API接口的开发，并支持动态数据交互，例如用户评论、内容管理等功能。通过该目录，我计划实现一个完整的前后端分离的应用架构。

这些技术方案将以独立的目录并行存在于同一个仓库中，每个目录都将实现相同的网站功能，但使用不同的技术栈。这将有助于我深入学习和对比不同框架的特点，并灵活选择最合适的技术来支持网站的长远发展。

欢迎对本项目提出建议或意见，并持续关注后续的更新！
