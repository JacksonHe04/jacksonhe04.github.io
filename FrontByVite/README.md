# 部署流程

## 步骤1: 确认所有提交已推送
- 在本地仓库中，确保所有更改都已经提交并推送到远程仓库。
- 使用命令 `git status` 检查是否有未提交的更改。
- 使用命令 `git push origin <branch-name>` 将本地分支的更改推送到远程仓库。

## 步骤2: 切换到项目目录
- 打开终端或命令行工具。
- 使用命令 `cd FrontByVite` 切换到项目的根目录。

## 步骤3: 执行部署脚本
- 在项目根目录下，运行命令 `npm run deploy`。
- 该命令将自动执行以下操作：
  - 构建项目并生成 `dist` 目录。
  - 将 `dist` 目录的内容推送到 `gh-pages` 分支。
  - 完成部署，无需手动构建。

## 注意事项
- 确保在执行 `npm run deploy` 前，已经安装了所有必要的依赖。可以使用 `npm install` 命令来安装。
- 如果遇到权限问题，请确保你有权限推送至远程仓库的 `gh-pages` 分支。
